# プログラムの構成
- Javaのプログラムはクラスの集まり。クラス名とファイル名は合わせる。クラスの内容を{ から }の間ブロックに書く。
- 処理のまとまりをメソッドと呼び、Javaはmainメソッドから実行される
- 処理文;(セミコロン)で終わる。
    - System.out.println()は、()内のデータを表示する。
``` java
class クラス名 {
   public static void main(String[] args) {
        処理文;
    }
}
```

# 基本データ型の型変換
- 数値同士の演算や代入をするときに、データ型の変換が
    - 自動で行われる     : 表現できる値の幅が小→大
    - キャストで明示する  : 表現できる値の幅が大→小
Javaにおける基本データ型の型変更には、主に2つの方法があります：暗黙の型変換（アップキャスト）と明示的な型変換（ダウンキャスト）。以下にその違いと具体的な例を示します。

### 1. 暗黙の型変換（アップキャスト）
これは、小さいデータ型から大きいデータ型への変換を意味します。Javaが自動的に型変換を行います。

例:
```java
int myInt = 10;
double myDouble;

myDouble = myInt;  // intからdoubleへの暗黙の型変換

System.out.println(myDouble);  // 10.0と出力される
```

上記の例では、`int`型の変数`myInt`が`double`型の変数`myDouble`に暗黙的に変換されました。

### 2. 明示的な型変換（ダウンキャスト）
これは、大きいデータ型から小さいデータ型への変換を意味します。このとき、データの損失が発生する可能性があるため、明示的に変換する必要があります。

例:
```java
double myDouble = 10.5;
int myInt;

myInt = (int) myDouble;  // doubleからintへの明示的な型変換

System.out.println(myInt);  // 10と出力される（小数部分は切り捨てられる）
```
上記の例では、`double`型の変数`myDouble`が`int`型の変数`myInt`に明示的に変換されました。このとき、`myDouble`の小数部分が切り捨てられています。
これらの型変換の際には、データの損失やオーバーフローなどの問題が発生しないよう注意が必要です。特に、明示的な型変換を行う際には変換前後のデータ型の範囲をよく理解しておくことが重要です。 

# 構造化プログラミング
構造化プログラミングは、プログラムをより理解しやすく、効率的に、そして信頼性の高いものにするためのプログラミングパラダイムの一つです。1970年代初頭に提唱され、ソフトウェアの品質を向上させるための手法として広く受け入れられました。

以下の主な原則に基づいています：

1. **シーケンス（順序）**
    - ステートメントは、指定された順序で上から下へと実行される。

2. **選択（if/else）**
    - 条件に基づいて、2つ以上のステートメントシーケンスの中から1つを選択して実行する。

3. **繰り返し（ループ）**
    - ある条件が満たされるまで、ステートメントシーケンスを繰り返し実行する。

これらの原則を使うことで、GOTO文（無条件のジャンプ命令）の使用を避けることができ、コードの流れが予測しやすくなります。

### 具体例

#### 1. シーケンス
```java
System.out.println("ステップ1: エンジンをスタートする。");
System.out.println("ステップ2: シートベルトを締める。");
System.out.println("ステップ3: 車を運転する。");
```
上記のコードは、指定された順序で3つの命令を実行します。

#### 2. 選択
```java
int age = 20;
if (age >= 18) {
    System.out.println("あなたは成人です。");
} else {
    System.out.println("あなたは未成年です。");
}
```
上記のコードは、`age`の値に基づいて2つのうちの1つのメッセージを出力します。

#### 3. 繰り返し
```java
for (int i = 1; i <= 5; i++) {
    System.out.println("繰り返し回数: " + i);
}
```
上記のコードは、メッセージを5回出力します。

これらの基本的な構造を組み合わせることで、より複雑なタスクを解決するプログラムを構築できます。構造化プログラミングの目的は、コードの読みやすさ、保守性、および信頼性を向上させることにあります。

# else if
`else if`文は、複数の条件を順序立ててチェックする場合に使用します。基本的な`if`文と`else`文だけでは、2つの選択肢しか持てませんが、`else if`を使用することで、3つ以上の選択肢を持つことができます。

### `else if`文の使用例

考えられるシナリオ：学生の得点に基づいてグレードを決定する場面を考えます。

- 90点以上は"A"
- 80点以上は"B"
- 70点以上は"C"
- 60点以上は"D"
- それ未満は"F"

以下はこのシナリオをJavaで実装した例です。

```java
int score = 85;  // ここでは85点としていますが、任意の点数を設定できます。

if (score >= 90) {
    System.out.println("あなたのグレードはAです。");
} else if (score >= 80) {
    System.out.println("あなたのグレードはBです。");
} else if (score >= 70) {
    System.out.println("あなたのグレードはCです。");
} else if (score >= 60) {
    System.out.println("あなたのグレードはDです。");
} else {
    System.out.println("あなたのグレードはFです。");
}
```

このコードを実行すると、「あなたのグレードはBです。」と出力されます。

### 仕組みの解説：

1. 最初の`if`文で、`score`が90以上かどうかを確認します。
2. もし90以上ではない場合、次の`else if`文で、`score`が80以上かどうかを確認します。
3. これを続けて、最後にどの条件にも当てはまらない場合の`else`文へと進みます。

このように`else if`文を使用することで、複数の条件を順にチェックし、それぞれの条件に対して異なるアクションを実行することができます。

# switch
`switch`文は、変数の値に基づいて複数のアクションの中から1つを選択して実行する制御文です。`if-else if-else`文と同様の役割を果たすことができますが、`switch`文の方が特定のシナリオでは読みやすくなることがあります。

- 式はbyte,short,int,charのみ
- 式の値によって処理の開始位置が決まる
- breakは{}を抜ける

### `switch`文の基本的な構文：

```java
switch (変数) {
    case 値1:
        // 変数が値1のときに実行するコード
        break;
    case 値2:
        // 変数が値2のときに実行するコード
        break;
    // ... 他のcase文 ...
    default:
        // 変数が上記のいずれの値にも一致しないときに実行するコード
}
```

### 具体例：

曜日の番号（1～7）を文字列（"月曜日"～"日曜日"）に変換するシナリオを考えてみます。

```java
int dayOfWeek = 4;  // ここでは4（木曜日）としていますが、1から7までの任意の数字を設定できます。

switch (dayOfWeek) {
    case 1:
        System.out.println("月曜日");
        break;
    case 2:
        System.out.println("火曜日");
        break;
    case 3:
        System.out.println("水曜日");
        break;
    case 4:
        System.out.println("木曜日");
        break;
    case 5:
        System.out.println("金曜日");
        break;
    case 6:
        System.out.println("土曜日");
        break;
    case 7:
        System.out.println("日曜日");
        break;
    default:
        System.out.println("正しい番号を入力してください。");
}
```

このコードを実行すると、「木曜日」と出力されます。

### 注意点：
- 各`case`の後には`break`ステートメントが必要です。これは、一致する`case`のコードが実行された後、`switch`文を終了するためのものです。`break`を省略すると、次の`case`のコードも実行される「フォールスルー」が発生します。
- `default`節はオプションですが、変数の値がどの`case`にも一致しない場合に実行されるコードを指定するために使用されます。

# while文
`while`文は、指定した条件が真である限り、中のブロックのコードを繰り返し実行する制御文です。具体的には、`while`文の条件が真（`true`）である場合、ループの中身が実行され、それが終わると再び条件がチェックされます。このプロセスが条件が偽（`false`）になるまで繰り返されます。

### `while`文の基本的な構文：

```java
while (条件) {
    // 条件が真の場合に実行されるコード
}
```

### 具体例：

以下は、数字を1から5まで印刷するシンプルな例です。

```java
int number = 1; // 開始値

while (number <= 5) {  // numberが5以下である間、ループを続ける
    System.out.println("Number: " + number);
    number++;  // numberの値を1増やす
}
```

このコードを実行すると、以下の出力が得られます：

```
Number: 1
Number: 2
Number: 3
Number: 4
Number: 5
```

### ポイント：

- `while`ループの中で条件がいつ偽になるかを管理する変数や条件を適切に設定することが重要です。そうしないと、ループが無限に繰り返される「無限ループ」が発生する可能性があります。
- 上記の例では、`number++`によって`number`の値が各ループの実行後に1ずつ増加しています。この操作によって、いずれ条件`number <= 5`が偽になり、ループが終了します。

`while`文は、事前に繰り返しの回数が決まっていない場面や、特定の条件が満たされるまでの間繰り返す必要がある場面で非常に役立ちます。

# for文
もちろんです！

`for`文は、指定した回数だけ繰り返し処理を行う制御文です。一般的な使用シナリオは、コレクションのすべての要素を処理する場面や、指定した回数だけ特定の操作を繰り返す場面などです。

### `for`文の基本的な構文：

```java
for (初期化; 条件; ステップ) {
    // 条件が真の場合に実行されるコード
}
```

- **初期化**：ループの最初に1回だけ実行される式。多くの場合、ループ用の変数の初期化に使用されます。
- **条件**：この条件が真である限り、ループは続行されます。
- **ステップ**：各ループの反復後に実行される式。多くの場合、ループ用の変数の更新に使用されます。

### 具体例：

以下は、数字を1から5まで印刷する例です。

```java
for (int i = 1; i <= 5; i++) {
    System.out.println("Number: " + i);
}
```

このコードを実行すると、以下の出力が得られます：

```
Number: 1
Number: 2
Number: 3
Number: 4
Number: 5
```

### 解説：

1. `int i = 1;` で、変数 `i` を1で初期化します。
2. `i <= 5` という条件をチェックします。初めてのループでは `i` が1なので、この条件は真です。
3. 条件が真である場合、ループの中のコードが実行されます。
4. `i++` で、`i` の値が1増加します。
5. 2-4のステップが、条件が偽になるまで繰り返されます。

このように、`for`文は、ループの初期化、条件チェック、およびステップを1つの文にまとめて書くことができるため、特定の回数の繰り返し処理を行う場合に非常に便利です。
